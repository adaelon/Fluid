# Fluid 切片计划

> 配套 `docs/技术方案.md`。每刀独立可验、依赖文件状态而非会话记忆(A4)。验证用确定性工具(B2)。
> 验证基准项目:`E:\allwork\download\agent\alphaGPT`(33 个 `.py`、57 个非噪声文件,Python;含 `.understand-anything/` 图谱 91 节点/199 边)。

## 切片依赖序

```
S1 后端骨架 ─┬─ S2 图谱加载
             └─ S3 前端壳+CM6 ── S4 tree-sitter 解析
S5 缓存 ─── S6 单函数生成(后端) ── S7 流式生成+渲染 ─┬─ S7.5 生成状态反馈 ── S7.6 失败归因+重试
                                                      ├─ S8 调度+大文件
                                                      └─ S9 gutter+手动单行
S10 追问器 ── S11 视觉打磨(注释材质 ↔ IDE 壳一致性)
U 轨(界面壳,ADR-0015): U1 IDE 布局 ── U2 文件 tab+面包屑 ── U3 Open Folder 切文件夹 ── U4 命令面板(可选)
```

## S1 · 后端骨架 + L0 文件树
- **做**:Rust 后端(axum+tokio),`fluid ./project` 起服务;ProjectReader 列文件树、读源码;`GET /api/project/tree`、`GET /api/file`。
- **不做**:不碰图谱、不碰 LLM、不碰前端。
- **判据**:对 alphaGPT 起服务,`/api/project/tree` 返回文件树(57 文件,含 33 个 `.py`);`/api/file?path=execution/config.py` 返回源码;**无 `.understand-anything/` 也能起**。
- **触达**:`crates/fluid-server/`(`main.rs`、`project_reader.rs`、`routes.rs`、`Cargo.toml`)。

## S2 · 图谱加载(可选 + UTF-8 解码,GBK 兜底)
- **做**:GraphLoader 读 `knowledge-graph.json`,正确解码中文 summary,提供 nodes/edges 查询;`GET /api/project/graph`。
- **不做**:不改前端;不依赖图谱存在。
- **判据**:对 alphaGPT 解析出 91 节点/199 边,summary 中文不乱码;删除该目录后接口返回 `null` 且服务不崩(ADR-0011)。
- **触达**:`crates/fluid-server/src/graph_loader.rs`。

## S3 · 前端壳 + CM6 只读 + 文件树导航
- **做**:浏览器前端;文件树侧栏;点击文件 → CM6 只读渲染源码(真空态:gutter 无指示点)。
- **不做**:不解析、不生成、不显影。
- **判据**:浏览器打开,点 alphaGPT 任一文件,CM6 只读显示源码;追问器置灰显示"打开文件以启用追问"。
- **触达**:`web/`(Vue 3 + Vite,ADR-0014)、`web/src/FileTree.vue`、`web/src/Editor.vue`(CM6 包 `shallowRef`)。

## S4 · tree-sitter 解析(Python + Rust)
- **做**:tree-sitter WASM;对源码出 roster(FunctionSpan[])、lineRange、重点行 query。
- **不做**:不接 LLM;重点行规则先给 Python 一版、Rust 一版基础规则。
- **判据**:对 `execution/config.py` 与一个 `.rs` 样本,控制台输出函数清单 + 重点行行号,与人工核对一致。
- **触达**:`web/parser/`、`web/parser/keyline-queries/{python,rust}.scm`。
- **PENDING 消化**:重点行 query 具体规则在此刀定稿。

## S5 · 缓存(.fluid/,函数 span hash + 模型版本)
- **做**:CacheStore;`put/get`;键 = `hash(fnSpan)+modelVersion+promptVersion`(技术方案 §6)。
- **不做**:不接 LLM(用假数据测往返)。
- **判据**:put 后 get 命中且不调下游;改 fnSpan → miss;写入落在 `.fluid/`,源码不变。
- **触达**:`crates/fluid-server/src/cache_store.rs`。

## S6 · 单函数生成(后端,非流式)
- **做**:LLMProxy(持 key,模型 = opencode zen 网关的 **glm-5.1**,OpenAI 兼容)+ ContextAssembler;`/api/generate` 先做"整段返回";接 S5 缓存。
- **不做**:暂不流式、不前端渲染。
- **判据**:对 alphaGPT 某函数返回 Capsule + 重点行注释 JSON(中文、合理);**二次调用命中缓存零 token**(日志断言无 LLM 调用)。
- **触达**:`crates/fluid-server/src/{llm_proxy,context_assembler,routes}.rs`。

## S7 · 流式生成 + 前端渲染(显影)
- **做**:`/api/generate` 改 WS 流式(capsule→lines);前端 GhostStore + CM6 block widget 渲染胶囊/行级;§7.2/7.3 显影动画;折叠/展开纯 UI。
- **不做**:暂不做调度排序(顺序生成即可)、暂不大文件降级。
- **判据**:打开文件,函数胶囊逐个滑出、重点行卡片显影;折叠再展开**零重算**(日志断言);关文件释放内存、重开命中缓存秒显。
- **触达**:`web/ghostStore.ts`、`web/render/capsuleWidget.ts`、`web/render/lineWidget.ts`。

## S7.5 · 生成状态反馈(已完成 2026-06-15)
- **做**:函数级"生成中"骨架占位(`capsule` 帧到即替换)+ 编辑区全局进度 chip(`done/total`,基于每函数 `done`/`error` 帧计数),完成态短暂提示后淡出。
- **不做**:不动后端;不做 gutter 指示点(S9);不做 IDE 状态栏(U1,chip 是临时落点);不做调度并发(S8)。
- **判据**:打开 py/rs 文件即见每函数"生成中"骨架 + "生成中 x/N";capsule 到达替换真卡片、计数前进;全部完成"✓ N 已显影"并淡出。**npm run build 绿**(详见 `代码链路.md` S7.5)。
- **触达**:`web/src/render/placeholderWidget.ts`、`ghostStore.ts`、`render/ghostField.ts`、`Editor.vue`、`styles.css`。
- **源起**:用户反馈"生成静默无提示,用户很懵"。

## S7.6 · 失败归因 + 重试(已完成 2026-06-15)
- **做**:后端两个错误分支加日志(归因可见:`LLM error` 网关 / `LLM parse error` 坏 JSON);前端失败处显示「· 生成失败」+ 原因 tooltip + 「重试」按钮;`retry(fnId)` 经现有 WS 重发单函数(socket 断则重连补发 pending)。
- **不做**:不做自动重试/退避(坏 JSON 会无限烧 token);不加请求超时(reqwest 仍无 timeout,真挂起停在"生成中",留后续)。
- **判据**:`cargo test` 31/31、`npm run build` 绿;失败处出现「重试」点击重发(详见 `代码链路.md` S7.6)。
- **触达**:`crates/fluid-server/src/routes.rs`、`web/src/render/{placeholderWidget,ghostField}.ts`、`ghostStore.ts`、`Editor.vue`、`styles.css`。
- **源起**:用户问"为什么有些函数失败" + "给失败处加重试"。

## S8 · 生成调度 + 大文件视口渐进
- **做**:GenerationScheduler:视口邻近排序、有限并发(3-5)、滚动重排;超阈值退视口渐进(占位"滚动到此处自动生成")。
- **不做**:阈值数值先取默认(>20 函数/>800 行),实测再调。
- **判据**:埋点验证生成顺序为视口邻近;大文件打开只生成视口批,滚动触发其余。
- **触达**:`web/scheduler.ts`。
- **PENDING 消化**:大文件阈值在此刀定。

## S9 · gutter 指示 + 手动单行补
- **做**:gutter 空心微点(函数顶)/ 实心脉动点(重点行);非重点行 hover → "解释这一行" → `/api/explain-line`。
- **判据**:重点行有脉动点;点非重点行触发单行生成并显影;单行结果入缓存。
- **触达**:`web/render/gutter.ts`、`crates/fluid-server/src/routes.rs`(explain-line)。

## S10 · 追问器(分层上下文 + 跨文件 + 临时取源)
- **做**:QueryPanel UI(§6.3 材质);`/api/query` 流式;分层装配 + 超窗降级(ADR-0006);跨文件一句话摘要注入(ADR-0007/0011);临时取源破例。
- **判据**:对某函数追问得答案;问跨文件被调函数实现 → 触发临时取源且事后目标文件仍真空;构造大文件追问不爆窗(走降级)。
- **触达**:`web/src/QueryPanel.vue`、`crates/fluid-server/src/context_assembler.rs`(query 分支)。

## S11 · 视觉打磨(材质一致性)
- **做**:玻璃态统一渲染管线、语义色温竖线、折叠 4px 边线、文件切换 <150ms 色温过渡;§7.4 铁律核对。
- **判据**:逐条对照 §7 视觉规范;胶囊与行级注释无圆角/阴影/字体差异。
- **触达**:`web/styles/`、`web/render/*`。

## U 轨 · 类 VSCode 界面壳(ADR-0015)

> 外层 IDE chrome 走标准视觉;代码区内注释保留玻璃态(§7.4 适用范围已收窄)。

### U1 · IDE 布局骨架
- **做**:活动栏 + 资源管理器侧栏 + 编辑区 + 底部状态栏;面板可拖宽;S7.5 进度 chip 迁入状态栏。
- **不做**:不做多 tab(U2)、不做切文件夹(U3)。
- **判据**:三区布局 + 状态栏出现;拖动分隔条改宽;进度显示在状态栏。
- **触达**:`web/src/App.vue`、新增 `web/src/shell/`(布局组件)、`styles.css`。

### U2 · 文件 tab + 面包屑
- **做**:多文件 tab(打开/激活/关闭)、路径面包屑;切 tab = 切激活文件(复用 Editor 激活链)。
- **判据**:开多个文件成多 tab,点 tab 切换,关 tab 释放;面包屑显示当前路径。
- **触达**:`web/src/shell/Tabs.vue`、`App.vue`。

### U3 · Open Folder 切文件夹
- **做**:后端加"开文件夹"端点(运行时换根 / 多根)+ 前端选择器;切根后刷新文件树。
- **不做**:不做最近打开列表(后续)。
- **判据**:从 UI 选另一个目录 → 文件树切换 → 打开新目录文件正常生成;路径穿越防护对新根仍生效。
- **触达**:`crates/fluid-server/src/{routes,project_reader}.rs`、`web/src/api.ts`、`shell/`。
- **PENDING 消化**:换根 vs 多根的取舍在此刀定。

### U4 · 命令面板(可选)
- **做**:Ctrl+P 快速打开文件 / Ctrl+Shift+P 命令面板(折叠全部、重试全部失败等)。
- **判据**:Ctrl+P 模糊找文件并打开。
- **触达**:`web/src/shell/CommandPalette.vue`。

## 验收锚点(全程,B2 确定性验证)
- 未打开文件调任意接口 → `INACTIVE_CONTEXT` 且无 LLM 调用(自动化断言)。
- 重开未变文件 → 缓存命中、零 token(日志断言)。
- 折叠/展开 → 零重算(日志断言)。
- 源码字节数/mtime 全程不变(零字节污染断言)。
