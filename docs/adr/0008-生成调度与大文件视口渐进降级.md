# 生成调度按视口邻近,大文件退回视口渐进生成

打开文件后,函数胶囊生成按**视口邻近度**排序、**有限并发**(3-5 在途),用户滚动即重排优先级——让注意力所在处最先凝结。小文件(阈值内)打开即全量 eager 生成;**大文件(超阈值,如 >20 函数或 >800 行)退回视口渐进生成**:只生成视口邻近批,其余占位"滚动到此处自动生成",进入视口时再生成。

## Considered Options

- **大文件也无闸全量生成**:打开 2000 行文件瞬间烧大量 token,而用户可能只看一个函数,与省 Token 底色冲突。
- **超阈值时弹确认/预算提示**:防误伤但打断体验。

## Consequences

- 这是对 ADR-0002「文件=原子激活」的 **size-based 让步**:大文件的激活不再是"整文件 eager",而是"视口渐进"。换言之文件内对超大文件恢复了按需,以控成本。
- 已生成函数的折叠/展开仍是纯 UI 零重算(ADR-0002 不变)。
- 阈值具体数值留待实现期依实测成本/延迟调参。
