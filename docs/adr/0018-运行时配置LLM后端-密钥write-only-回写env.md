# 运行时配置 LLM 后端:面板可改三项,密钥 write-only 不回显,回写 .env

LLM 后端(`OPENCODE_API_KEY` / `OPENCODE_BASE_URL` / `FLUID_LLM_MODEL`)此前只能在启动前经 `.env`/环境变量配置、改一次须重启(`main.rs` 启动读一次,`LlmProxy` 与 `model` 存进 `AppState` 后不可变)。本 ADR 加一个**设置面板(U5)**让用户运行时改这三项:保存后**热重建内存 `LlmProxy` 立即生效**,**并回写 fluid 仓库根的 `.env`** 使重启保留。三项里 `api_key` 在面板里可输入——这跨越了既有「密钥永不发给前端」的安全姿态(`llm_proxy.rs` 注释),用户(2026-06-16)明确知悉并接受,缓释手段是 **write-only**:后端 `GET` 永不回显完整 key,只给 masked 状态(`···`+末 4 位)与「已配置/未配置」;面板输入框留空即保持原 key,填了才覆盖。仅支持 OpenAI 兼容 `/chat/completions`(现有抽象,够覆盖切 provider/模型/key 的主诉求),不引入多协议适配。

## Considered Options

- **密钥落盘位置 —— 只存内存(重启回落 .env)**:面板改动不新增任何落盘点,最保守;但每次重启须重填或改 .env,持久性差。否决。
- **密钥落盘位置 —— 独立用户级配置(`~/.config/fluid/`)**:跨项目共享一份配置;但要新增一套「文件 vs env 谁优先」的加载/优先级逻辑。选**回写 .env**:key 本就明文在已 gitignore 的 `.env` 里(落盘姿态不恶化),复用既有机制、零新增加载路径。
- **密钥暴露 —— GET 回显完整 key**:面板打开即可见可校对;但每次开面板 key 都从后端出到浏览器,暴露面最大。选 **write-only**:key 仅单向写入,GET 只给 masked 状态,把「key 进前端」限定为必要的写方向。
- **入口形态 —— 状态栏 dock / 资源管理器侧栏视图**:dock 与追问器争底部空间;侧栏视图依赖尚未建的活动栏多视图切换。选**活动栏底部齿轮 → 居中模态**:VSCode 惯例位置,表单不挤主区,关闭零占位。
- **协议范围 —— 支持任意 LLM 协议(含非 OpenAI 形)**:覆盖面广;但不同请求/响应形需各写适配,且 `LlmProxy` 全链路(生成/追问/SSE 解码)硬绑 OpenAI 兼容。否决,留真有需求再开。

## Consequences

- **安全姿态变更(有意、有界)**:`api_key` 经本地 HTTP(127.0.0.1)进入浏览器**写方向**。原铁律「密钥永不进前端」放宽为「只写不回显」;「永不写入服务项目 `.fluid/`、永不提交」**不变**(回写的是 fluid 仓库的 `.env`,已 gitignore,非服务项目目录)。
- **`AppState` 可变化**:`llm: Option<LlmProxy>` 与 `model: String` 由不可变改为锁后可换(沿 `project: RwLock<ProjectCtx>` 既有模式),使运行时热换成立。
- **model ↔ 缓存键耦合(ADR-0003)**:`model` 进缓存键,故换 model 时须**重建 cache 指针**(`CacheStore::new(root, model, prompt)`,镜像 `open_folder` 的换根重建);旧 model 下已生成的注释因分区不同不再命中,重开文件按新 model 重新生成——符合预期、非 bug。
- **`.env` 回写需保留其余内容**:回写逻辑为纯函数(更新已存的三行、追加缺失的、不动其他行/注释),可确定性单测(B2)。
- **拓扑不变(ADR-0010)**:仍只有 `LlmProxy` 出网;设置面板是本地后端的配置入口,不新增出网点。
- **CONTEXT.md 不变**:设置面板是 UI/基建实现件,非领域术语,不进术语表。
- **新增 U5 切片**(见 `切片计划.md`):U5a 后端运行时配置(端点 + 锁化 + .env 回写 + 热换)、U5b 前端设置面板(活动栏齿轮 + 居中模态 + write-only key)、U5c(可选)测试连接。与计划内 **U4=命令面板** 是两个不同功能(术语冲突已澄清,本功能另开 U5)。
