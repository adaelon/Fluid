# 前端整体壳定为类 VSCode 界面,玻璃态材质收窄为仅约束注释

前端整体界面在「自研全屏玻璃态 UI」与「类 VSCode IDE 壳」之间选 **类 VSCode 壳 + 代码区内保留玻璃态注释** 的混合方案。外层 chrome(活动栏 / 资源管理器 / 文件 tab / 面包屑 / 可拖宽面板 / Open Folder / 状态栏 / 命令面板)走标准 IDE 视觉;代码区内的函数胶囊与行级幽灵注释**继续**用 `需求文档 §7.2/7.3` 的显影动画与 `§7.4` 玻璃态材质连续性。判断基准:用户(2026-06-15)看 S7 demo 后明确反馈现自研三栏界面"太丑",要熟悉、专业的 IDE 观感,但不愿丢掉 Fluid 注释显影的特色。由此 **`§7.4` 玻璃态铁律的适用范围从"整个 UI"收窄为"代码区内的注释材质"**——这是本 ADR 的第二个决策,与壳的选型不可分。

## Considered Options

- **类 VSCode 完整壳(全部标准 IDE 视觉,含代码区)**:最熟悉,但会一并抹掉胶囊/行级注释的玻璃态显影——Fluid 的核心视觉特色,用户明确要保留,否决。
- **保留自研全屏玻璃态 + 只清理粗糙**:不动 `§7` 设计语言,仅做间距/对齐/字体打磨。改动最小,但仍是"自成一派的非标准 IDE 观感",不解决用户的核心诉求(陌生、不专业),否决。
- **混合:VSCode 壳 + 玻璃态注释(选中)**:外层熟悉、专业,内层保留特色。代价是两套视觉语言需在边界处调和(壳的冷峻 IDE 灰 vs 注释的玻璃态),且需重写 `需求 §7` 的适用范围。用户在三选项中选此项。

## Consequences

- **`需求文档 §7` 改写适用范围**:`§7.4` 材质连续性铁律(无圆角/无阴影/胶囊与行级同字体)**只约束代码区内的幽灵注释**,不再约束外层壳;外层壳采用类 VSCode 标准视觉。`§7.1` 真空态、`§7.2/7.3` 显影、`§7.5` 折叠态语义不变,均限定在"注释"语境。
- **新增前端 U 轨切片**(见 `切片计划.md`):U1 IDE 布局骨架(活动栏 + 资源管理器 + 编辑区 + 状态栏,可拖宽)、U2 文件 tab + 面包屑、U3 Open Folder 切文件夹(后端加开文件夹端点 + 前端选择器)、U4 命令面板(可选)。原 S11 视觉打磨重定义为"注释材质 ↔ IDE 壳的最终一致性核对"。
- **后端新增能力(U3)**:`ProjectReader` 当前启动即 canonicalize 单根并绑定路径穿越防护;Open Folder 需要运行时换根(或多根)+ 一个开文件夹端点。这是 ADR-0010 拓扑下的增量,不改"仅 LLMProxy 出网 / 解析骨架本地"的边界。
- **临时件迁移**:S7.5 的进度 chip(现浮于编辑区右上)在 U1 落地后迁入底部状态栏。
- **CONTEXT.md 不变**:活动栏 / tab 等是 UI 实现件,非领域术语,不进术语表。
- 框架(ADR-0014 Vue 3)、编辑器内核(ADR-0012 CM6)、拓扑(ADR-0010)均不变;本 ADR 仅落定外层壳的视觉方向与玻璃态铁律的适用边界。

## 修订(2026-06-15,U3-R:Open Folder 选择器机制)

U3 落地时选**文本框输绝对路径**,理由记为"本地后端架构下浏览器无法把服务端路径交给后端,故非系统文件选择器"。用户眼验后反馈要"点击 → 弹文件管理器 → 确定"。

- **修订**:加 `POST /api/project/pick`,由**本地后端**(= 用户本机,ADR-0010)弹原生文件夹对话框(`rfd` crate)→ 返回选中的服务端绝对路径 → 前端喂现成 `/api/project/open` 换根。文本框降为兜底。
- 原决策的约束(浏览器拿不到绝对路径)**仍成立**;本修订是**绕过**它(让后端而非浏览器开对话框),不是推翻它。
- 命门:rfd 跨平台依赖树大(仅按 Windows target fetch 才精简);同步对话框在 macOS 须主线程事件循环(Windows 任意线程可,现可用)。详见 `docs/代码链路.md` U3-R。
