# 后端语言定为 Rust

本地后端用 Rust(`axum` + `tokio`)。后端是个"文件 + LLM 代理 + 缓存"的瘦代理——解析已在浏览器端(ADR-0009),后端不碰 AST,职责仅:HTTP/WS 服务、读文件、文本解码(`encoding_rs`,UTF-8 优先、GBK 仅非法 UTF-8 兜底)、解析 understand-anything JSON(`serde_json`)、LLM 流式代理(`reqwest`)、`.fluid/` 缓存。Rust 全栈成熟,性能非瓶颈(LLM 延迟主导),且契合团队深耕 Rust 与项目要支持 Rust 语言。

## Considered Options

- **Node + TypeScript**:`npx fluid` 分发零摩擦、前后端天然共享 TS 类型;但这两点优势有限——前端无论如何是 TS(CM6 是 JS 库),类型共享可由 ts-rs 抹平。

## Consequences

- **分发**:不再是 `npx`,改为各平台预编译 binary(`fluid ./project`)或 `cargo install`,多一步安装。
- **类型共享**:用 `ts-rs` 从 Rust 类型自动生成前端 TS,避免双份定义。
- **仓库形态**:前端 TS + 后端 Rust 的多语言仓库(前端本就是 TS,影响有限)。
- 拓扑不变(ADR-0010 仍成立),仅后端实现语言落定。
