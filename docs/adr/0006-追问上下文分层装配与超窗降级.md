# 追问上下文分层装配,超窗逐级降级

追问以当前文件为上下文边界,但不把整份源码硬塞进 prompt。默认装配 = 文件级 summary + 全部函数胶囊摘要 + edges + focus(聚焦函数/行)的完整源码与注释——即整个文件以"摘要粒度"在场保证全局视野,只有聚焦处以"源码粒度"放大。这样上下文体积随文件大小是摘要级增长而非全源码增长,天然有界。

## Considered Options

- **整文件全源码硬塞**:小文件可行,但大文件(数千行/数十函数)直接超 LLM 上下文窗口,且信噪比差;只能配合"文件大小上限 + 超限拒绝",体验割裂。

## Consequences

- 当全部函数胶囊摘要之和仍超窗(超大文件)时,**再降一级**:只保留 focus 邻近函数的源码/摘要,远端函数仅留名称。
- 跨多个未聚焦函数实现细节的追问可能答不准 → 提供"模型按需追加某函数源码"的回退,补源码后再答一轮。
