# 部署拓扑:本地后端 + 浏览器前端

Fluid 以"对着本地项目目录跑的本地工具"形态交付(形如 `npx fluid ./project`):本地起一个后端进程,读项目源码与 `.understand-anything/`、持有 LLM key 并做 LLM 代理、把缓存写进项目内 `.fluid/`;浏览器前端负责 Monaco/CodeMirror 渲染与 tree-sitter WASM 解析。契合 understand-anything 本身就是在项目目录里产出 `.understand-anything/` 的本地 CLI 工作流。

## Considered Options

- **纯浏览器 + File System Access API**:LLM key 无法安全持有,FSA 权限体验一般,缓存只能进 OPFS/IndexedDB(其他工具看不到,且 ADR-0003 的 `.fluid/` 落盘不成立)。
- **桌面 App(Electron/Tauri)**:体验最像 IDE 但打包分发更重;暂不选,保留为后续可能。

## Consequences

- 文件读写、API key、`.fluid/` 落盘缓存都有合法安放处;ADR-0003 的项目内落盘成立。
- LLM 调用一律经本地后端代理,key 不进前端。
- 出网边界清晰:仅 LLM 生成/追问出网,解析与缓存命中全本地。
- 代价:用户需本地安装/启动进程。
