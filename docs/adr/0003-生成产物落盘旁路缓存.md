# 生成产物落盘旁路缓存,键 = 内容 hash + 模型版本

已生成的函数胶囊与行级注释落盘到源码之外的旁路缓存文件,缓存键 = 文件内容 hash + 模型版本。重开文件命中缓存则零 Token 秒显;文件内容或模型变更时 hash 失效,自动重算。写旁路文件不动源码,核心律 1「零字节污染」不破。

## Considered Options

- **需求文档 §4.3/§5 原方案(纯内存 + 基于 gitCommitHash 的硬缓存)**:understand-anything 输出的 `gitCommitHash` 为空字符串,硬缓存无法建立,实际效果是每次重开文件都重新 LLM 全量生成、重复烧 Token——与"省 Token"动机矛盾。

## Consequences

- 用文件内容 hash 绕开坏掉的 gitCommitHash。
- 更换 prompt 模板或模型版本会导致缓存整体失效重算(可接受)。
- 缓存落盘位置(如 `.understand-anything/fluid-cache/` 或 `.fluid/`)留待实现期定。
