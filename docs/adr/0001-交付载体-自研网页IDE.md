# 交付载体定为自研网页 IDE

Fluid 需要在代码行之间内联渲染毛玻璃语义卡片、自定义 gutter 脉动点与显影动画(需求文档 §7)。VS Code / JetBrains 插件的装饰 API 无法支持这类自由内联富渲染,故选择基于 Monaco 或 CodeMirror 自研一个只读网页 IDE,换取对渲染管线的完全控制。

## Considered Options

- **VS Code 插件**:decoration API 不能在行间插入带 `backdrop-filter` 的富卡片,gutter 仅能放图标,§7 视觉规范九成无法实现。
- **JetBrains 插件**:Inlay Hints 可做行内组件,但仍难还原玻璃态材质,且开发栈为 JVM/Kotlin。

## Consequences

代价是等于自造一个轻量 IDE,工作量最大。Monaco vs CodeMirror 的二选一留待实现期决定。
