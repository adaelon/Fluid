# 文件为原子激活单元,放弃三级惰性链

零 Token 契约的粒度定在**文件级**:未打开的文件绝对真空(零数据零 Token),一旦打开文件即一次性生成该文件全部函数胶囊 + 重点行注释。需求文档 §4 原设计的 L0→L1(函数摘要)→L2(展开函数才出行级、受视口±5 闸门)三级惰性链就此压成两级,函数展开/折叠退化为纯 UI 显隐,不再触发生成或销毁。

## Considered Options

- **保留三级惰性链 + 严守 §9 <200ms/<500ms 首屏**:在"函数/行级注释绝大多数必须 LLM 现场生成"的现实下(understand-anything 只给稀疏函数摘要、零行级数据),LLM 按秒计,首屏指标物理上不可达,二者不可兼得。

## Consequences

- 选可信赖的成本契约,**放宽** §9 首屏延迟指标,改为按函数流式显影(见 ADR-0004)。
- 需求文档 §4.1 / §4.2 / §4.3 / §6.2 / §9 须重写,详见 `docs/需求文档修订清单.md`。
- 文件内不再有"函数级按需";行级仍保留长尾的手动单行补(用户主动"解释这一行")。
