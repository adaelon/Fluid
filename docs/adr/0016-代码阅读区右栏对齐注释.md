# 代码阅读区行级注释改为右侧栏对齐,函数胶囊改为单行横头

代码阅读区的行级幽灵注释从「块卡片插在被注释行上方」改为「右侧注释栏、与对应代码行 1:1 对齐」(陪读式);函数胶囊从多字段块卡片改为函数定义行上方的**单行横头**(`name · 摘要 · 复杂度`)。当某行注释比单行代码高时,**该代码行的行高随注释增长**——代码文字靠行顶、注释在右栏同一横带内换行,既保证严格对齐又保证全文默认可见。判断基准:用户(2026-06-15)在 ADR-0015 选定类 VSCode 壳后进一步反馈,U 轨重点不在外层 chrome,而在「代码阅读区本身的美观与可读性」;现行模型把卡片插在代码行之间,纵向撑开代码、打断连贯扫读,是可读性的主要矛盾。另注:现行 CM6 未挂任何暗色主题(`basicSetup` 自带浅色高亮),在 `#0d1117` 深色壳内对比刺眼,本 ADR 一并修复。

## Considered Options

- **保持行间块卡片,仅打磨材质/主题**:改动最小、不动架构,但「代码被卡片撑开、扫读被打断」这一核心痛点不解决,否决。
- **注释默认折叠,hover/点击召出**:代码最纯净,但 Fluid「打开即显影、注释默认可见」的产品特色被弱化,否决。
- **右栏单行截断 + hover 看全文**:对齐零漂移,但中文一句话常超一行 → 默认看不全,理解代码要额外悬停,否决。
- **右栏对齐 + 行高随注释增长(选中)**:代码列不被打断、注释始终可见且严格 1:1 对齐;代价 = 重写 §7.3 放置模型、长注释所在行变高、需调和右栏与追问器的右缘归属。用户在候选中选此项。

## Consequences

- **需求 §7 改写**:§7.2 函数胶囊由多字段卡片改为单行横头(仍在定义行上方,保留语义凝结核与淡入显影);§7.3 行级显影由「块卡片在行上方」改为「右侧栏对齐、行高随注释增长、左侧语义色温竖线、从右淡入+微滑」;§7.5 折叠语义改为「点胶囊头 → 该函数右栏行注释隐藏、胶囊头收为细色线」(仍仅隐不删、零重算)。§7.1 真空态不变;§7.4 材质铁律不变,且**继续约束右栏内注释**。
- **暗色主题**:前端为 CM6 挂一套匹配现有 `#0d1117` GitHub-dark 壳色的自写 HighlightStyle(不引 one-dark,避免撞现有调色板)。
- **CM6 渲染机制**:行注释 = inline widget(`side:1`)挂重点行行尾 + 该行 `padding-right` 留出右栏带 + widget 绝对定位落进右栏;inline widget 的高度撑高所在行,实现「行高随注释增长」。胶囊 = block widget(`side:-1`)在定义行上方。右栏随 CM6 行布局共享滚动,免滚动同步。
- **新增切片**(见 `切片计划.md` U-R 轨):U-R1 右栏重构(暗色主题 + 行注释移右栏 + 胶囊横头)、U-R2 字体大小可调。**U-R 轨排在 IDE chrome(U1~U4)之前**(用户重定优先级)。
- **PENDING — 追问器右缘碰撞**:QueryPanel 现固定右下(`styles.css:.query-panel position:fixed`),与右栏争右缘;U-R1 暂让追问器浮于 rail 之上,正式 dock 留到 S10 追问器实做时重评。
- **CONTEXT.md 不变**:「右侧注释栏」「字体大小调节」均为 UI 实现件,沿 ADR-0015 先例不入术语表。
- 框架/内核/拓扑(ADR-0010/0012/0014/0015)不变;本 ADR 仅落定代码阅读区注释的放置模型与暗色主题方向。

## 修订(2026-06-15,U-R1 眼验后)

用户实看 U-R1 后修正:行注释**不要共享固定右栏列**(短代码行的注释被甩到最右、离代码太远,且所有注释左缘对齐成一列显得割裂),改为**尾随式 —— 每条注释紧跟其代码行末尾、固定间距,各行起点不同**;且**函数横头与行注释过长都要自适应换行,不截断**。

- 放置由「`float:right` 固定宽右栏列 + `.cm-line` flow-root 撑高」改为「inline widget 挂行尾 + CSS `inline-block`(`margin-left` 固定间距、`max-width` 限宽换行、`vertical-align:top` 撑高行)」。inline-block 天然参与行盒高度,**不再需要 flow-root / 连续分隔线**(均移除)。
- 「代码列不被打断 + 行高随注释增长 + 代码靠行顶」三条核心承诺**不变**;变的只是注释横向落点(共享列 → 尾随各行)。
- 命门更新:短行注释贴近、长行注释随代码右移(需横向滚动看全),这是「尾随代码」的固有取舍,用户接受。
