## 1. 拓扑与技术栈

```
$ npx fluid ./project
┌─────────────────── 浏览器前端 (CodeMirror 6) ───────────────────┐
│  FileTree(L0) · CM6 只读渲染 · tree-sitter WASM 解析             │
│  GhostStore(内存) · GenerationScheduler · QueryPanel(追问器)     │
└───────────────▲───────────────────────────── WS/HTTP ──────────┘
                │ source / skeleton / 流式生成 / 追问
┌───────────────┴──────────── 本地后端 (Rust) ────────────────────┐
│  ProjectReader(fs) · GraphLoader(.understand-anything,可选)      │
│  ContextAssembler · LLMProxy(持 key) · CacheStore(.fluid/)       │
└──────────────────────────────────────────────────────────────────┘
        出网边界:仅 LLMProxy 出网;解析/骨架/缓存命中全本地
```

- **前端**:渲染 + 解析 + 调度 + 内存态幽灵注释。无 key、不落盘。
- **后端**:文件 IO、图谱加载、上下文装配、LLM 代理、`.fluid/` 缓存。
- **后端技术栈**:Rust + `axum`/`tokio`(HTTP/WS)、`serde_json`、`encoding_rs`(非 UTF-8 兜底解码;图谱实测为合法 UTF-8)、`reqwest`(LLM 流式代理)、`ts-rs`(类型生成 TS)。分发:各平台预编译 binary(`fluid ./project`),非 npx。
- **前端技术栈**:Vue 3(SFC)、TypeScript、CodeMirror 6、tree-sitter WASM、Vite。CM6 实例与 GhostStore 等命令式大对象用 `shallowRef`/`markRaw` 持有,不进 Vue 响应式(ADR-0014)。
- 依据:ADR-0009(浏览器解析)、ADR-0010(拓扑)、ADR-0012(CM6)、ADR-0013(后端 Rust)、ADR-0014(前端 Vue 3)。

## 3. 核心数据类型

```typescript
// L0 骨架
type FileNode   = { path: string; name: string; lang: 'py' | 'rs' | 'other' };
type Edge       = { source: string; target: string; type: 'calls'|'imports'|...; };
type GraphNode  = { id: string; type: 'file'|'class'|'function'|'document';
                    filePath: string; summary: string; lineRange?: [number, number] };

// tree-sitter 解析产物(前端)
type FunctionSpan = { id: string; name: string; lineRange: [number, number] };
type FileParse    = { roster: FunctionSpan[]; keyLines: Map<fnId, number[]> };

// 生成产物(幽灵注释)
type Capsule        = { fnId: string; signature: string; summary: string;
                        complexity: string; io: string };
type LineAnnotation = { fnId: string; lineNumber: number; text: string; color: string };
```

## 4. 对外接口(B1 签名)

```typescript
// — L0 骨架 —
GET  /api/project/tree            -> { files: FileNode[] }
GET  /api/project/graph           -> KnowledgeGraph | null      // 可选增强

// — 文件激活:源码 + 骨架增强(前端据此 tree-sitter 解析)—
GET  /api/file?path=...           -> { source: string;
                                       fileSummary?: string;     // 无图谱则 LLM 现补
                                       edges?: Edge[] }

// — 按函数流式生成(WebSocket)—
WS   /api/generate
  ⇢ { reqId, filePath, fn: FunctionSpan, roster: string[], keyLines: number[],
      shared: { fileSummary?, edges?, calleeSummaries?: Record<string,string> } }
  ⇠ { reqId, kind: 'cache-hit' }                 // 命中:零 token,后随完整产物
  ⇠ { reqId, kind: 'capsule', capsule: Capsule }
  ⇠ { reqId, kind: 'line', line: LineAnnotation }
  ⇠ { reqId, kind: 'done' | 'error' }

// — 手动单行补(长尾非重点行)—
POST /api/explain-line { filePath, fn, lineNumber } -> LineAnnotation

// — 追问(WebSocket,分层上下文)—
WS   /api/query
  ⇢ queryInline { sessionId, filePath, focus?, userQuery, previousTurnId? }
  ⇠ token 流 -> { answer, confidence }

// 闸门:文件未激活 → 任意接口返回 INACTIVE_CONTEXT(§8.2)
```

## 5. 生成管道(后端伪代码)

```
generateCapsule(req):
    key = hash(sourceOf(req.fn)) + MODEL_VERSION
    if cache.has(key):
        emit 'cache-hit'; emit cache.get(key); emit 'done'    # 零 token
        return
    ctx = assembleGenContext(req)
    stream = LLMProxy.generate(prompt(req.fn, ctx))            # 流式
    buf = {}
    for chunk in stream:
        emit chunk            # 'capsule' / 'line' 逐个推前端显影
        buf.merge(chunk)
    cache.put(key, buf)       # -> .fluid/
    emit 'done'

assembleGenContext(req):
    return {
      fileSummary: req.shared.fileSummary ?? LLM_summarizeFile(req.filePath),
      roster:      req.roster,                                 # 本文件函数清单(ADR-0004)
      edges:       req.shared.edges ?? treeSitterEdges(req.filePath),
      calleeSummaries: for callee in crossFileCallees(req.fn):  # ADR-0007/0011
          graphSummary(callee) ?? cachedOneLiner(callee) ?? LLM_oneLine(callee)
    }
```

## 6. 缓存键(对 ADR-0003 的细化)

- **粒度 = 函数**(非整文件):键 = `hash(函数源码 span) + 模型版本 + prompt 版本`。
- **收益**:文件内某函数改动只失效该函数,其余函数缓存仍命中,重算最小化。
- **行级注释**随所属函数胶囊一同存取(同一缓存条目)。
- **存储**:`.fluid/capsules/<key>.json`,源码零字节不动(核心律 1)。
- > 说明:此处把 ADR-0003 的"文件内容 hash"细化为"函数 span hash",以支持部分失效;语义一致,粒度更细。

## 7. 前端激活状态机

```
VACUUM ──open file──> LOADING ──source+skeleton 到达──> PARSING(tree-sitter)
PARSING ──roster+keyLines──> ACTIVATING
  ACTIVATING: scheduler 按视口邻近排队;有限并发发 /api/generate
              每函数:capsule 到 → 渲 block widget;line 到 → 渲行级 widget
  小文件:全量入队   大文件:仅视口邻近批入队,余者占位(ADR-0008)
ACTIVATED ⇄ scroll:重排队列 / 大文件进入视口即入队
ACTIVATED ──fold/unfold──> 纯 UI 显隐,GhostStore 不变,零重算
ACTIVATED ──close file──> VACUUM(释放 GhostStore;.fluid/ 缓存留存)
```

## 8. 追问上下文装配(分层,ADR-0006)

```
buildQueryContext(filePath, focus):
    base = fileSummary + allCapsuleSummaries + edges          # 全文件摘要级
    base += fullSource(focus.fn) + annotations(focus)          # 聚焦处源码级
    if tokens(base) > BUDGET:
        base = neighborFns(focus) only;  远端函数仅留名         # 降一级
    crossFile = calleeSummaries(focus.fn)                      # 跨文件一句话
    return base + crossFile
    # 回退:模型请求"追加某函数源码" → 补该函数源码再答一轮
    # 破例:问未打开文件被调函数实现 → 临时取源 span,用完即弃(ADR-0007)
```

## 9. 关键约束回指

| 约束 | 落点 |
|---|---|
| 源码零字节污染 | 缓存/注释全在内存或 `.fluid/`,§6 |
| 未打开文件零 token | 文件级闸门,§7 状态机 VACUUM |
| 重开命中零 token | §5 cache-hit 分支 |
| 折叠/展开零重算 | §7 fold = 纯 UI |
| 大文件不烧/不爆窗 | §7 视口渐进 + §8 分层降级 |
