**PENDING 定稿 — 重点行规则**:行位于**函数体内**且首节点 ∈ {赋值(`assignment`/`let_declaration`/`assignment_expression`)、有值 `return`、`raise`/`assert`、分支/循环/with 头(`if`/`for`/`while`/`match`/`with`)、语句位调用、Rust 块尾调用(隐式返回)}。**跳过**:import/use、装饰器/属性宏、`def`/`class`/`fn`/`impl` 结构头、`pass`、try-except 脚手架头(其内部语句仍算)。类级/模块级行不挂任何函数胶囊 → 非重点行。**v1 已知局限**:Rust 非调用型尾表达式(如 `&self.root`)不算重点行。

**入口**:`FluidParser.create([{lang,grammarWasm,keyLineQuery}])` → `parser.parse('py'|'rs', source)`。当前仅被验证脚本调用;CM6/GhostStore 接线在 S7。

## 2026-06-14 S5 · 旁路缓存(.fluid/,函数 span hash + 模型/prompt 版本)

**触达**(后端新增缓存层,不接 LLM/前端 — ADR-0003、技术方案 §6):
- `crates/fluid-server/src/cache_store.rs:{Capsule,LineAnnotation,CapsuleEntry}` — 生成产物类型(技术方案 §3,camelCase serde rename);一个缓存条目 = 一函数胶囊 + 其行级注释(同条存取,§6)。

- `crates/fluid-server/src/cache_store.rs:fnv1a_step` + `FNV_OFFSET/FNV_PRIME` — 内联 FNV-1a 64,键 = `hex(fnv1a(model_version ⊕ prompt_version ⊕ fnSource))`,字段间 NUL 分隔防拼接混叠。
- `crates/fluid-server/src/routes.rs:AppState.cache` — 注入 `CacheStore`(`#[allow(dead_code)]`,S6 消费)。
- `crates/fluid-server/src/main.rs` — `mod cache_store`(模块级 allow,同 S4 未接线惯例);构造 `CacheStore::new(reader.root(), MODEL_VERSION_PLACEHOLDER, PROMPT_VERSION)` 注入 AppState。`PROMPT_VERSION="p1"`;**模型版本 = `s5-unset` 占位**(选型 PENDING → S6,不预写死)。

**入口**:`AppState.cache.put/get`;当前仅被 `cache_store::tests` 调用,接线到 `/api/generate` 在 S6。

**测试**(确定性 B2,5 条,自清理 TempDir,全套 14/14 绿):
- `put_then_get_hits_and_round_trips` — 冷查 miss→put→get 命中且产物逐字节往返。
- `changed_fn_span_misses` — 改一字节 fnSpan→key 变→miss。
- `model_or_prompt_version_change_invalidates` — 同源改 model/prompt 版本→miss(ADR-0003 失效)。
- `writes_under_dot_fluid_and_leaves_source_untouched` — 落盘在 `.fluid/capsules/*.json`;旁路文件源码 bytes+mtime 不变(核心律 1)。
- `corrupt_entry_reads_as_miss` — 损坏 JSON 读作 miss。

> **hash 算法 = 内联 FNV-1a 64(非 std DefaultHasher,非外部 crate)**:磁盘持久键需跨进程/平台/工具链稳定,`DefaultHasher` 不保证;FNV 算法固定即稳定,零依赖,合本项目"自写小工具"惯例(S1 自写 TempDir)。64 位非密码学,碰撞→错命中,但单项目函数 span 规模下概率可忽略。**何时回头**:若碰撞为患,换 SHA-256(改算法仅令缓存全失效重算,代价低)。未开 ADR,因 reverse 成本 = 缓存重算,不满足 ADR 三条门槛。
